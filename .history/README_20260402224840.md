# Quran Live M3U8 - HLS Streaming Server

A lightweight HLS (HTTP Live Streaming) server for broadcasting Quran audio files with continuous looping, multi-listener synchronization, and YouTube audio extraction capabilities. Built with Express.js and FFmpeg.

## ✨ Features

### Core Streaming
- 🎵 **Continuous Looping** - Seamless infinite playback with `-stream_loop -1`
- 👥 **Multi-Listener Sync** - All listeners stay synchronized to the same stream position
- ⚡ **Low Resource Usage** - Direct stream copy (no re-encoding) uses < 5% CPU
- 📡 **HLS Streaming** - Generates `.m3u8` playlists and `.ts` segments automatically
- 🔄 **Auto-Restore** - Streams automatically restart after server reboot

### Advanced Features
- 🎬 **YouTube Audio Streaming** - Extract and stream audio from YouTube videos with authentication support
- 🔐 **Cookie Authentication** - Download age-restricted or private content using browser cookies
- 🎭 **Role-Based Access Control** - Admin (full access) and User (view-only) roles
- 📊 **Dual Logging System** - Separate console and file logging with configurable levels
- 🌐 **CORS Enabled** - Works with external players and applications
- 📈 **Real-Time Diagnostics** - Monitor streams, FFmpeg processes, and system health

## System Requirements

### Minimum Requirements
- **OS**: Linux (Ubuntu/Debian recommended) or Windows
- **CPU**: 1 core (single-core sufficient due to low resource usage)
- **RAM**: 256MB minimum, 512MB recommended
- **Storage**: 1GB free space (depends on number of audio files)
- **Node.js**: v14.x or higher
- **FFmpeg**: v4.0 or higher

### Recommended for Production
- **OS**: Ubuntu 20.04 LTS or Debian 11
- **CPU**: 2+ cores
- **RAM**: 1GB+
- **Network**: 10 Mbps+ upload bandwidth per concurrent stream

### Logging Levels

The application supports dual logging with independent levels:

- **ERROR** - Critical errors only
- **WARN** - Warnings and errors
- **INFO** - General information (default console level)
- **DEBUG** - Debug details
- **VERBOSE** - Everything (default file level)

**Example:**

```bash
FILE_LOG_LEVEL=VERBOSE

CONSOLE_LOG_LEVEL=DEBUG
FILE_LOG_LEVEL=DEBUG
```

Logs are written to `./logs/app.log` by default with timestamps and automatic rotation.

## Usage

### Starting the Server

```bash
# Production mode
node server.js

# Development mode (with auto-reload)
npm run dev
```

Server will start on `http://localhost:8300`

### Uploading Audio Files

#### Method 1: Web Dashboard (Admin Only)

1. Open `http://localhost:8300` in your browser
2. Click "Login" and enter admin credentials
3. Select an MP3 file and enter a stream name
4. Click "Upload & Start Stream" - the stream will start automatically

#### Method 2: API Request (Requires Admin Auth)

```bash
curl -X POST http://localhost:8300/api/upload \
  -u admin:@!JKF3eWd12 \
  -F "audio=@/path/to/quran.mp3" \
  -F "name=al-nas"
```

Response:
```json
{
  "success": true,
  "message": "Stream created successfully",
  "streamUrl": "http://localhost:8300/streams/al-nas/stream.m3u8"
}
```

### YouTube Audio Streaming

Extract and stream audio from YouTube videos (requires yt-dlp):

#### Installation:
```bash
# Install yt-dlp (recommended over youtube-dl)
pip install yt-dlp
# or
sudo apt install yt-dlp
```

#### Using Web Interface:
1. Navigate to `/youtube-streamer.html`
2. Enter stream name and YouTube URL
3. (Optional) Provide cookies.txt path for authenticated downloads
4. Click "Start Stream"

#### Using API:
```bash
# Without authentication
curl -X POST http://localhost:8300/api/streams/youtube/my-stream \
  -H "Content-Type: application/json" \
  -d '{"url":"https://www.youtube.com/watch?v=VIDEO_ID"}'

# With cookie authentication (for age-restricted/private videos)
curl -X POST http://localhost:8300/api/streams/youtube/my-stream \
  -H "Content-Type: application/json" \
  -d '{
    "url":"https://www.youtube.com/watch?v=VIDEO_ID",
    "cookies":"/root/cookies.txt"
  }'
```

**Cookie Authentication:**
To download age-restricted or private videos, export your browser cookies using a browser extension (e.g., "Get cookies.txt") and upload the file to your server. The system validates Netscape HTTP Cookie format automatically.

## Technical Details

### HLS Configuration

The server generates HLS streams with optimized settings:

- **Segment Duration**: 3 seconds (low latency)
- **Playlist Size**: 6 segments (~18 seconds buffer)
- **Loop Mode**: Infinite (`-stream_loop -1`)
- **Input Mode**: Real-time (`-re`)
- **Codec**: Direct copy (no re-encoding, `-c:a copy`)
- **Format**: MPEG-TS (`.ts` segments)
- **Flags**: Auto-delete old segments, round durations
- **Video**: Disabled (`-vn`) for audio-only streams

### FFmpeg Command Structure

**For Local Audio Files:**
```bash
ffmpeg -re -stream_loop -1 -i input.mp3 \
  -vn \
  -c:a copy \
  -hls_time 3 \
  -hls_list_size 6 \
  -hls_flags delete_segments+round_durations \
  -hls_segment_filename seg_%03d.ts \
  stream.m3u8
```

**For YouTube Streams:**
```bash
# Step 1: Extract audio using yt-dlp
yt-dlp --extract-audio --audio-format mp3 \
  --audio-quality 192K \
  --cookies /root/cookies.txt \
  -o "audio.mp3" \
  "https://www.youtube.com/watch?v=VIDEO_ID"

# Step 2: Stream extracted audio via FFmpeg (same as above)
```

### Resource Usage

Typical resource consumption per stream:
- **CPU**: 2-5% (single core, no re-encoding)
- **RAM**: 20-50 MB
- **Disk I/O**: Minimal (sequential writes)
- **Network**: 128-320 kbps (depends on audio bitrate)

### Logging Architecture

Logs are written in structured format with timestamps:

**Console Output (INFO level by default):**
```
[2024-01-15T10:30:00.000Z] [INFO] Starting stream: al-nas
[2024-01-15T10:30:01.000Z] [INFO] FFmpeg started with PID: 12345
```

**File Output (VERBOSE level by default):**
```
[2024-01-15T10:30:00.000Z] [VERBOSE] Input file: /path/to/audio.mp3
[2024-01-15T10:30:00.100Z] [DEBUG] Checking write permissions...
[2024-01-15T10:30:00.200Z] [INFO] Stream started successfully
```

Log files are located at `./logs/app.log` (configurable via `LOG_FILE_PATH`).

## Troubleshooting

### Dashboard Shows "Login Required" for Actions

**Problem**: Cannot start/stop/delete streams, buttons are disabled

**Solution**:
1. Click the "Login" button in the top-right corner
2. Enter admin credentials (username: `admin`, password: `@!JKF3eWd12`)
3. The UI will update to show admin controls

### YouTube Download Fails

**Problem**: Error when trying to stream from YouTube

**Solutions:**

1. **Install yt-dlp:**
   ```bash
   pip install yt-dlp
   # or
   sudo apt install yt-dlp
   ```

2. **Check video availability:**
   ```bash
   yt-dlp -F "https://www.youtube.com/watch?v=VIDEO_ID"
   ```

3. **Use cookies for restricted content:**
   - Export browser cookies using extension (e.g., "Get cookies.txt")
   - Upload cookies.txt to server: `scp cookies.txt root@server:/root/`
   - Provide path in YouTube streamer interface

4. **Verify cookies format:**
   ```bash
   head -5 /root/cookies.txt
   # Should show: # Netscape HTTP Cookie File
   ```

### FFmpeg Files Not Created

**Problem**: `stream.m3u8` or `.ts` files not appearing

**Solution**:
1. Check write permissions on `streams/` folder:
   ```bash
   chmod -R 755 streams/
   chown -R $(whoami):$(whoami) streams/
   ```

2. Verify FFmpeg is installed and in PATH:
   ```bash
   which ffmpeg
   ffmpeg -version
   ```

3. Check logs for detailed error messages:
   ```bash
   tail -f logs/app.log
   ```

4. For YouTube streams, check if video has audio-only format available

### 404 Errors When Accessing Streams

**Problem**: HTTP 404 when requesting `/streams/[name]/stream.m3u8`

**Solution**:
1. Verify the stream exists:
   ```bash
   curl http://localhost:8300/__debug/streams
   ```

2. Check if FFmpeg process is running:
   ```bash
   curl http://localhost:8300/__debug/ffmpeg
   ps aux | grep ffmpeg
   ```

3. Ensure case-sensitive filename matches (Linux is case-sensitive)

4. Check stream status via API:
   ```bash
   curl http://localhost:8300/api/streams
   ```

### High CPU Usage

**Problem**: FFmpeg using excessive CPU (>20%)

**Solution**:
1. Verify stream copy mode is enabled (check logs for `-c:a copy` and `-vn`)
2. Reduce number of concurrent streams
3. Use lower bitrate audio files
4. Check if video is being processed (should have `-vn` flag)

### Stream Not Looping

**Problem**: Stream stops after playing once

**Solution**:
1. Check FFmpeg command includes `-stream_loop -1`
2. Verify `-re` flag is present (prevents speed-up)
3. Restart the stream via API or dashboard
4. Check logs for FFmpeg exit codes

### Permission Denied Errors

**Problem**: Cannot write to streams folder or create log files

**Solution**:
```bash
# Fix ownership
sudo chown -R $USER:$USER streams/ logs/

# Fix permissions
chmod -R 755 streams/ logs/

# Verify
ls -la streams/ logs/
```

### Multi-Listener Desync

**Problem**: Different listeners hear different parts of the stream

**Solution**:
1. Ensure all clients connect to the SAME playlist URL
2. Clear player cache/buffer
3. Use players that support HLS live sync (VLC, hls.js)
4. Consider using a CDN for large audiences

### Logging Issues

**Problem**: Too much/too little output in console or logs

**Solution**:
1. Adjust logging levels in `.env`:
   ```bash
   CONSOLE_LOG_LEVEL=INFO    # Reduce console output
   FILE_LOG_LEVEL=VERBOSE    # Keep detailed file logs
   ```

2. Check log file location:
   ```bash
   cat logs/app.log | tail -50
   ```

3. Verify log directory exists and is writable:
   ```bash
   ls -la logs/
   ```

## Deployment

### Running as a Service (systemd)

Create `/etc/systemd/system/quran-stream.service`:

```ini
[Unit]
Description=Quran HLS Streaming Server
After=network.target

[Service]
Type=simple
User=www-data
WorkingDirectory=/opt/quran-live-m3u8
ExecStart=/usr/bin/node server.js
Restart=always
Environment=NODE_ENV=production
Environment=PORT=8300

[Install]
WantedBy=multi-user.target
```

Enable and start:
```bash
sudo systemctl daemon-reload
sudo systemctl enable quran-stream
sudo systemctl start quran-stream
sudo systemctl status quran-stream
```

### Nginx Reverse Proxy Setup

Install Nginx:
```bash
sudo apt install nginx
```

Configure `/etc/nginx/sites-available/quran-stream`:
```nginx
server {
    listen 80;
    server_name your-domain.com;

    location / {
        proxy_pass http://localhost:8300;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
    }

    # Optimize for HLS streaming
    location /streams/ {
        proxy_pass http://localhost:8300/streams/;
        proxy_cache off;
        proxy_buffering off;
        add_header Cache-Control no-cache;
        
        # CORS headers
        add_header Access-Control-Allow-Origin *;
        add_header Access-Control-Allow-Methods 'GET, OPTIONS';
    }
}
```

Enable site:
```bash
sudo ln -s /etc/nginx/sites-available/quran-stream /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl reload nginx
```

## Security Considerations

1. **Change Default Password**: Edit `middleware/auth.js` to set a strong password
2. **Use HTTPS**: Deploy behind Nginx with SSL/TLS certificate
3. **Firewall Rules**: Only expose necessary ports (80/443)
4. **Regular Updates**: Keep Node.js and FFmpeg updated
5. **Monitor Logs**: Check for unauthorized access attempts

## Performance Optimization

### For Multiple Concurrent Listeners

1. **Enable Nginx caching** for HLS segments
2. **Use a CDN** for geographic distribution
3. **Increase playlist size** for better buffering
4. **Optimize network MTU** for TS packet delivery

### For Multiple Simultaneous Streams

1. **Limit concurrent streams** based on CPU capacity
2. **Use separate output directories** for I/O isolation
3. **Monitor resource usage** via diagnostic endpoints
4. **Consider horizontal scaling** with load balancer
