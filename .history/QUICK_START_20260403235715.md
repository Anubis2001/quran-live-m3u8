# 🚀 Quick Start Guide - Authentication System

## Getting Started in 5 Minutes

### 1️⃣ Install Dependencies
```bash
npm install
```

### 2️⃣ Configure Environment
Edit `.env` file and add:
```env
SESSION_SECRET=your-random-secret-key-here
DB_HOST=localhost
DB_USER=root
DB_PASSWORD=
DB_NAME=quran_streaming
```

### 3️⃣ Start the Server
```bash
npm start
```

### 4️⃣ Access the Application
Open your browser and go to:
```
http://localhost:8300
```

## 🔑 Basic Operations

### **Creating a New User**
1. Go to `/register.html`
2. Enter username (3-30 characters, letters/numbers/underscores)
3. Enter email (optional but recommended)
4. Create password (minimum 8 characters)
5. Confirm password
6. Click "Create Account"

### **Logging In**
1. Go to `/login.html`
2. Enter username and password
3. Check "Remember me" if desired
4. Click "Sign In"
5. You'll be redirected to the dashboard

### **Resetting Password**
1. Go to `/forgot-password.html`
2. Enter your registered email
3. Click "Send Reset Link"
4. Use the token provided (in production, check your email)
5. Go to reset password page and enter new password

### **Managing Users (Admin Only)**
1. Login as admin
2. Click "Manage Users" button in header
3. From the admin panel you can:
   - Create new users
   - Edit existing users
   - Change user roles (user/admin)
   - Suspend/activate accounts
   - Delete users

## 🔧 API Usage Examples

### **Register a New User**
```javascript
fetch('/api/auth/register', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    username: 'newuser',
    email: 'user@example.com',
    password: 'SecurePass123'
  })
})
.then(res => res.json())
.then(data => console.log(data));
```

### **Login and Get Token**
```javascript
fetch('/api/auth/login', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    username: 'admin',
    password: '@!JKF3eWd12'
  })
})
.then(res => res.json())
.then(data => {
  console.log('Token:', data.token);
  // Store token for future requests
  localStorage.setItem('authToken', data.token);
});
```

### **Make Authenticated Request**
```javascript
const token = localStorage.getItem('authToken');

fetch('/api/streams', {
  headers: {
    'Authorization': `Bearer ${token}`
  }
})
.then(res => res.json())
.then(data => console.log(data));
```

### **Change Password**
```javascript
fetch('/api/auth/change-password', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
    'Authorization': `Bearer ${token}`
  },
  body: JSON.stringify({
    currentPassword: 'OldPassword123',
    newPassword: 'NewSecurePass456'
  })
})
.then(res => res.json())
.then(data => console.log(data));
```

## ❓ Troubleshooting

### **"Database connection failed"**
The app works without MySQL but authentication features won't persist. To fix:
1. Install MySQL
2. Create database: `CREATE DATABASE quran_streaming;`
3. Update `.env` with your MySQL credentials
4. Restart server

### **"Authentication required" errors**
1. Clear browser cookies
2. Refresh the page
3. Login again

### **"Rate limit exceeded"**
Wait 15 minutes or restart the server (in development).

### **Can't access admin pages**
Make sure you're logged in as the `admin` user, not a regular `user`.

## 🔒 Security Tips

### **For Production:**
1. ✅ Change default admin password immediately
2. ✅ Set strong `SESSION_SECRET` in `.env`
3. ✅ Use HTTPS (SSL certificate)
4. ✅ Configure proper `ALLOWED_ORIGINS`
5. ✅ Keep Node.js and dependencies updated
6. ✅ Monitor logs for suspicious activity
7. ✅ Regular database backups
8. ✅ Use strong MySQL password

### **Password Best Practices:**
- Minimum 8 characters (enforced)
- Mix of uppercase, lowercase, numbers, symbols
- Don't reuse passwords across sites
- Change passwords regularly
- Use a password manager
